## 1. 快速入门

### 小红是什么？

小红是安幕诺家族的 AI 股票交易辅助系统。她每天自动完成：

- 🌅 **盘前分析**：扫描全球指数、北向资金、板块热点
- 🔍 **盘中监控**：实时资金流向、持仓异动
- 🌹 **策略信号**：5 大策略（均线/RSI/MACD/布林/组合共识）
- 🛡️ **风控管理**：R 值仓位计算、止损止盈跟踪
- 📊 **交易统计**：胜率、盈亏比、PnL 曲线

### 三种使用方式

| 方式 | 适用人群 | 特点 |
|------|---------|------|
| 📱 微信小程序 | 移动端用户 | 随时随地查看信号和持仓 |

| 🔌 REST API | 开发者 / 量化 | 程序化调用，策略集成 |

## 4. API 使用指南

### 基础信息

```
Base URL:  https://api.xiaohong.family  (或 http://localhost:8000)
认证方式:  Bearer Token 或 X-API-Key Header
格式:      JSON
```

### 获取 Token

```bash
curl -X POST https://api.xiaohong.family/auth/login \
  -H "Content-Type: application/json" \
  -d '{"api_key": "xh-your-key-here"}'

# 返回:
# {"token": "eyJ...", "tenant_id": "default", "expires_in": 86400}
```

### 常用接口

#### 行情

```bash
# 全球指数
curl https://api.xiaohong.family/market/index \
  -H "X-API-Key: xh-your-key"

# 个股行情
curl https://api.xiaohong.family/market/quote/600519

# 北向资金
curl https://api.xiaohong.family/market/north-flow

# 板块排名
curl "https://api.xiaohong.family/market/sectors?top_n=5"
```

#### 交易

```bash
# 下单（Paper Trading）
curl -X POST https://api.xiaohong.family/trade/order \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "symbol": "600519",
    "side": "BUY",
    "price": 1800,
    "quantity": 100,
    "stop_loss": 1710,
    "take_profit": 1980,
    "strategy_id": "MA-CROSS",
    "reason": "金叉买入"
  }'

# 查询持仓
curl https://api.xiaohong.family/trade/positions
```

#### 策略

```bash
# 获取信号
curl https://api.xiaohong.family/strategy/signals

# 策略列表
curl https://api.xiaohong.family/strategy/list
```

#### 风控

```bash
# R值仓位计算
curl "https://api.xiaohong.family/risk/r-value?price=100&stop_loss=95"

# 仓位检查
curl "https://api.xiaohong.family/risk/check/600519?price=1800"
```

#### 统计

```bash
# 交易统计
curl https://api.xiaohong.family/log/stats

# 按股票过滤
curl "https://api.xiaohong.family/log/stats?symbol=600519"

# 最近交易
curl "https://api.xiaohong.family/log/trades?limit=50"
```

### Python SDK 示例

```python
import requests

class XiaohongClient:
    def __init__(self, base_url, api_key):
        self.base = base_url
        self.headers = {"X-API-Key": api_key}
    
    def get_stats(self):
        return requests.get(f"{self.base}/log/stats", headers=self.headers).json()
    
    def place_order(self, symbol, side, price, qty, **kw):
        return requests.post(f"{self.base}/trade/order", json={
            "symbol": symbol, "side": side, "price": price, "quantity": qty, **kw
        }, headers=self.headers).json()

# 使用
client = XiaohongClient("https://api.xiaohong.family", "xh-your-key")
stats = client.get_stats()
print(f"胜率: {stats['win_rate']}%, 总盈亏: ¥{stats['total_pnl']:,.0f}")
```

## 6. 交易操作

### 下单流程

```
1. 策略引擎生成信号 → 
2. 风控系统检查 →
   ├─ 仓位上限 (单股≤33.3%)
   ├─ 持仓数量 (≤9只)
   ├─ 可用资金
   └─ R值风险计算
3. Paper Trading 执行（或券商API实盘）
4. 交易日志记录（CSV + SQLite + PostgreSQL）
5. 通知推送
```

### Paper Trading 模式

- **零风险模拟**：使用真实行情数据，虚拟资金交易
- **真实费用模拟**：佣金(万2.5) + 印花税(千1) + 过户费
- **滑点模拟**：买卖方向随机 0~0.3% 滑点
- **延迟模拟**：0.5~3.0 秒随机延迟
- **首次部署务必使用 Paper 模式验证策略**

### 仓位计算 (R 值)

```
R 值 = 总净值 × 凯利分数 ÷ 8
     = ¥898,698 × 0.125 ÷ 8
     = ¥6,927

建议股数 = R 值 ÷ (价格 × 100) × 100
         = ¥6,927 ÷ ¥1,800 × 100
         = 300 股
```

### 止损设置

- **固定止损**：技术面支撑位（如 MA20 下方 2%）
- **移动止盈**：盈利 ≥ 20% 启动，从最高点回撤 10% 平仓

## 7. 风控体系

### 核心参数

```
单股仓位上限:  33.3%
最大持仓数:    9 只
凯利分数:      0.125 (保守)
单笔最大亏损:  总净值 × 2%
止损方法:      技术面 R 系数止损
止盈方法:      移动止盈 (盈利20%启动)
```

| 级别 | 触发条件 | 响应 |
|:----:|---------|------|
| 🔴 P0 | 止损触发 / 仓位超限 | 立即平仓 |
| 🟡 P1 | 接近止损(2%内) / 单日亏损>3% | 准备减仓 |
| 🔵 P2 | 策略信号 / 报告生成 | 参考评估 |

### 弹药库风控报告

每天 15:30 自动生成风控报告，包含：
- 止损状态检查
- 仓位合规检查
- 持仓盈亏总览
- 移动止盈更新

## 12. 常见问题

### Q: 为什么北向资金显示为 0？

A: 北向资金数据来自 Tushare，是 T-1 日终数据。非交易日或数据源限流时显示为 0，交易日自动恢复。

### Q: Paper Trading 和实盘有什么区别？

A: Paper Trading 使用真实行情但在虚拟账户中模拟，零风险。实盘需要接入券商 API（QMT/MinisQMT），首次使用务必先充分验证 Paper Trading 策略表现。

### Q: 策略信号为什么很少？
