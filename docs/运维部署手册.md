## 3. 快速部署（Docker）

### 3.1 获取代码

```bash
cd /opt
git clone <repo-url> xiaohong
cd xiaohong
```

### 3.2 配置密钥

```bash
# 复制环境变量模板
cp .env.docker .env

# 编辑 .env，填入真实密钥
nano .env
```

**必填项**:
```bash
TUSHARE_TOKEN=你的Tushare_Token    # 从 https://tushare.pro 获取
PAPER_TRADING=true                  # 首次部署务必设为 true
```

**可选项**:
```bash
FEISHU_WEBHOOK_URL=                 # 飞书通知 webhook
SMTP_USER= / SMTP_PASSWORD=         # 邮件告警
WECHAT_APPID= / WECHAT_APPSECRET=   # 微信通知
```

### 3.3 构建并启动

```bash
# 构建镜像（首次约 5-10 分钟）
docker compose build

# 启动全部服务
docker compose up -d

# 查看状态
docker compose ps

# 预期输出:
# NAME                STATUS
# xiaohong-engine     Up
# xiaohong-cron       Up
# xiaohong-api        Up (healthy)
# xiaohong-redis      Up (healthy)
# xiaohong-postgres   Up (healthy)
```

### 3.4 验证部署

```bash
# 健康检查
curl http://localhost:8000/health

# 预期响应:
# {"status":"ok","version":"2.0.0","uptime":...,"tenants":1}

# 行情接口
curl http://localhost:8000/market/index

# 交易统计
curl http://localhost:8000/log/stats

# API 文档
# 浏览器打开: http://<服务器IP>:8000/docs
```

### 3.5 常用运维命令

```bash
# 重启
docker compose restart

# 查看日志
docker compose logs -f trading-engine
docker compose logs -f api-server

# 停止
docker compose down

# 完全清理（包括数据卷）
docker compose down -v
```

| 周六10:00 | 📈 周报分析 | (LLM prompt) | LLM | AI 深度周报 |

### 管理命令

```bash
# 查看全部 Cron
hermes cronjob action=list

# 暂停某任务
hermes cronjob action=pause job_id=<job_id>

# 恢复
hermes cronjob action=resume job_id=<job_id>

# 手动执行一次
hermes cronjob action=run job_id=<job_id>

# 修改调度
hermes cronjob action=update job_id=<job_id> schedule="0 9 * * 1-5"
```

### 非交易日处理

周末/节假日 Cron 会自动运行但报告会标注「非交易日」，业绩数据为空。无需手动暂停。

### 告警通道

| 级别 | 飞书 | 邮件 | 微信 | 短信 |
|:----:|:----:|:----:|:----:|:----:|
| P0 紧急 | ✅ | ✅ | ✅ | ✅ |
| P1 警告 | ✅ | ✅ | — | — |
| P2 信息 | ✅ | — | — | — |

### 告警触发条件

- **P0**: 止损触发、仓位超限 33.3%、API 断连超过 5 分钟
- **P1**: 接近止损线（距离 < 2%）、单日亏损 > 3%
- **P2**: 策略信号生成、定时报告完成、系统启动

## 9. 备份与恢复

### 备份

```bash
# 备份脚本
#!/bin/bash
BACKUP_DIR="/backup/xiaohong/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

# 数据库
docker compose exec postgres pg_dump -U xiaohong xiaohong > $BACKUP_DIR/db.sql

# 交易日志
cp workspace/transactions.csv $BACKUP_DIR/
cp data/transactions.db $BACKUP_DIR/

# 持仓
cp data/holdings.json $BACKUP_DIR/

# 配置（不含密钥）
cp .env.docker $BACKUP_DIR/

# 压缩
tar -czf $BACKUP_DIR.tar.gz $BACKUP_DIR
```

### 恢复

```bash
# 恢复数据库
docker compose exec -T postgres psql -U xiaohong xiaohong < backup/db.sql

# 恢复交易日志
cp backup/transactions.csv workspace/
cp backup/transactions.db data/
```

### 定时备份 Cron

```bash
# 每天凌晨 2 点备份
0 2 * * * /opt/xiaohong/scripts/backup.sh >> /var/log/xiaohong-backup.log 2>&1
```

## 10. 故障排除

### 常见问题

| 症状 | 原因 | 解决 |
|------|------|------|
| API 返回 500 | Tushare Token 过期/无效 | 检查 `.env` 中 `TUSHARE_TOKEN` |
| Docker 构建失败 | TA-Lib 编译问题 | `apt install libta-lib-dev` |
| 北向资金为 0 | 非交易日 / 数据源限流 | 正常现象，交易日自动恢复 |
| akshare RemoteDisconnected | 东方财富限流 | 等 10 分钟重试，自动 fallback |
| PostgreSQL 连接拒绝 | 容器未就绪 | `docker compose restart postgres` |
| Paper Trading 数据不更新 | 非交易时段 | A股 9:30-15:00 才有实时数据 |
| LED 灯带不亮 | I2C 未启用 | `sudo raspi-config` 启用 I2C |
| 仪表盘白屏 | API 未启动 | `docker compose up -d api-server` |

### 日志查看

```bash
# Python 引擎
docker compose logs -f trading-engine

# API 服务
docker compose logs -f api-server

# Cron
docker compose logs -f cron-scheduler

# 系统日志（Pi）
journalctl -u xiaohong-box -f --lines=100
```

### 重置 Paper Trading

```bash
# 清空模拟交易记录
rm workspace/transactions.csv
rm data/transactions.db
python3 -c "from transaction_logger import TransactionLogger; TransactionLogger()"
```

### 12.3 四级加载机制

```
发现阶段:  marketplace.json / skills-index.yaml  → 元数据
调用阶段:  SKILL.md 正文                           → 工作流指令
按需加载:  references/ 目录                        → 领域知识文档
手动执行:  scripts/ 目录                           → Python脚本
```

### 12.4 MCP ↔ Skill 桥接

Skills 定义"做什么、何时做"，MCP Tools 提供"数据获取+执行能力"。

```bash
# 技能调用 MCP 工具
a-share-market-regime → market-data-mcp.get_index_data
position-sizer-cn     → risk-mcp.calc_position_size
decision-officer      → strategy-mcp.get_signals + execution-mcp.place_order
```

映射文件: `mcp-bridge/skill-to-mcp.yaml`

### 12.5 工作流编排

```bash
# 每日盘前市场扫描（3步链式调用）
daily-market-scan:
  a-share-market-regime → watchtower-skill → risk-manager-cn(decision_gate)

# 盘中综合决策
daily-decision:
  risk-manager-cn → decision-officer → execution(decision_gate)

# 周度复盘
weekly-review:
  trade-journal-cn → post-market-review → risk-manager-cn(decision_gate)
```

### 12.6 Claude Code 集成


## 13. 多租户管理（v2.1 新增）

### 13.1 租户 CRUD

```bash
# 列出全部租户
python3 scripts/multi_tenant.py list

# 创建租户（返回明文API Key，仅显示一次！）
python3 scripts/multi_tenant.py create --id client-001 --name "客户A" --tier pro --days 365

# 轮换 API Key
python3 scripts/multi_tenant.py rotate --id client-001

# 停用租户
python3 scripts/multi_tenant.py deactivate --id client-001

# 验证 API Key
python3 scripts/multi_tenant.py verify --key xh-client-001-xxxx
```

### 13.2 租户等级与限额

| 等级 | 策略数 | 自选股 | 实盘 | 持仓上限 | 限流 |
|------|:--:|:--:|:--:|:--:|:--:|
| free | 1 | 5 | ✗ | 3 | 30/min |
| pro | 9 | 50 | ✓ | 15 | 100/min |
| family | 999 | 999 | ✓ | 50 | 500/min |
| internal | 9999 | 9999 | ✓ | 999 | 9999/min |

### 13.3 安全机制

- API Key → SHA-256 哈希存储 → 永不存明文
- 每个租户独立 JWT secret
- PostgreSQL schema-per-tenant 数据隔离
- 首次创建时返回明文 Key（仅一次）
