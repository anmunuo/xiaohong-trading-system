## 二、策略分类体系

```
策略生命周期：选股 → 入场 → 仓位管理 → 止损止盈 → 出场
             │                                        │
             └──────── 组合策略（编排以上全部）─────────┘
```

### 2.1 六大类别

| 类别 | 目录 | 核心职责 |
|:--|:--|:--|
| **选股（Selection）** | `strategies/selection/` | 从全市场/候选池中筛选优质标的 |
| **入场（Entry）** | `strategies/entry/` | 判断建仓时机，生成买入信号 |
| **仓位管理（Position）** | `strategies/position/` | 计算每笔交易应投入多少资金 |
| **止损止盈（Stop）** | `strategies/stop/` | 设置风险边界和利润保护 |
| **出场（Exit）** | `strategies/exit/` | 判断何时清仓离场 |
| **组合（Composite）** | `strategies/composite/` | 编排多个组件策略形成完整交易流水线 |

### 2.2 策略间数据契约

不同类别的策略通过标准化数据结构衔接：

```
SEL → 输出: [{code, name, score, current_price, ...}]
         ↓
ENT → 输入: stock_data ← SEL 输出的一项
     输出: {should_enter: bool, entry_price, confidence, ...}
         ↓
POS → 输入: stock_data + net_value
     输出: {position_value, shares, position_pct, ...}
         ↓
STP → 输入: stock_data + entry_price
     输出: {stop_loss_price, take_profit_price, ...}
         ↓
EXT → 输入: position_data
     输出: {should_exit: bool, reason, ...}
```

## 三、策略基类设计

### 3.1 继承体系

```
BaseStrategy (ABC)                         # 所有策略的根基类
├── SelectionStrategy : BaseStrategy       # 选股 → select()
├── EntryStrategy : BaseStrategy           # 入场 → should_enter()
├── PositionStrategy : BaseStrategy        # 仓位 → calculate_position()
├── StopStrategy : BaseStrategy            # 止损 → calculate_stop_loss() + calculate_take_profit()
├── ExitStrategy : BaseStrategy            # 出场 → should_exit()
└── CompositeStrategy : BaseStrategy       # 组合 → add_strategy() + execute()
```

### 3.2 BaseStrategy 核心能力

| 能力 | 方法 | 说明 |
|:--|:--|:--|
| 参数持久化 | `load_parameters()` / `save_parameters()` | JSON 文件 → `strategies/params/{ID}.json` |
| 参数校验 | `validate_parameters(params)` | 类型/范围/枚举检查 |
| 参数更新 | `update_parameters(new_params)` | 校验通过后合并+保存 |
| 参数模式 | `parameter_schema()` | 声明参数的类型/范围/默认值 |
| 性能追踪 | `record_execution(result)` | 累计执行次数/成功率/盈亏 |
| 性能报告 | `get_performance_report()` | 结构化性能摘要 |
| 回测 | `backtest(historical_data)` | 遍历历史数据执行策略 |
| 参数优化 | `optimize(historical_data, param_ranges)` | 网格搜索最优参数 |

### 3.3 参数管理模式

```
策略实例化
  │
  ├─ 1. default_parameters() → 提供合理默认值
  ├─ 2. load_parameters() → 从 JSON 文件覆盖默认值（文件优先）
  ├─ 3. validate_parameters() → 检查类型和范围
  └─ 4. 运行时 kwargs 可临时覆盖参数

策略进化
  │
  └─ 进化引擎 → update_parameters(new_params) → 保存到 JSON → 下次加载生效
```

**参数文件位置**：`strategies/params/{STRATEGY_ID}.json`

## 四、已实现策略详解

### 4.1 SEL-001 趋势跟随（选股）

**三因子评分体系**（权重：趋势40% + 量能30% + 价格30%）：

| 因子 | 分析方法 | 关键指标 |
|:--|:--|:--|
| 趋势 | 均线多头排列 + 线性回归斜率 + 高点突破 | MA5/MA10/MA20 |
| 量能 | 量比 vs 均量 + 量价配合 | 放量倍数 / 价升量缩检测 |
| 价格 | 价格位置分位 + 年化波动率 + 突破关键位 | 20日高低区间 / 年化σ |

**关键参数**：趋势周期(20日)、成交量倍数(1.5x)、评分阈值(60分)、最大选股数(10)

### 4.2 POS-002 R值风险管理（仓位）

**核心公式**：`仓位金额 = R值 ÷ 止损距离`

```
R值计算: 净值 × 单股最大持仓33.3% × 1/8 × 凯利值(0.2)
止损确定优先级: 用户提供 → 技术支撑位 → ATR波动率 → 默认5%
安全边际: 仓位打9折
上限检查: 不超过净值 × 33.3%
取整: 100股为单位
止盈: 盈亏比 3:1（止盈价 = 入场价 + (入场价 - 止损价) × 3）
```

**关键参数**：net_value(必传)、kelly_value(必传)、safety_margin(0.9)、min/max_stop_distance(3%/15%)

### 4.3 STP-001 R值止损（止损止盈）

**止损优先级**：技术支撑位 → 移动止盈 → 保本止损 → 时间止损 → 默认5%

```
移动止盈: 涨20%启动，从最高价回撤10% = 移动止损价
保本止损: 涨10%启动，止损上移至入场价 × 1.001
波动率调整: 高波动(>3%)股票自动扩大止损距离(×1.0~1.5)
板块调整: 半导体×1.4 / 新能源×1.3 / 科技×1.2 / 消费×1.0 / 金融×0.9
```

**关键参数**：trailing_start_pct(20%)、trailing_step_pct(10%)、break_even_pct(10%)、profit_target_ratio(3:1)

### 4.4 CMP-001 保守组合（编排策略）

将三个策略编排为完整流水线：

```
SEL-001 (40%) → POS-002 (30%) → STP-001 (30%)
```

**执行流程**：
1. `_execute_selection()` → 调用 SEL-001 从股票池选股
2. `_process_single_stock()` → 对每只选中股票计算仓位 + 止损
3. `_generate_recommendation()` → 综合评分排序，输出建仓建议

**风险控制**：最大持仓3只 / 单股上限33.3% / 最小现金比例20% / 绩效阈值0.6触发调整
