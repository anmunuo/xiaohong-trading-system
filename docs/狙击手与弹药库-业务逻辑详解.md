> 最后更新：2026-06-08 | 作者：小红 🌹

## 一、系统定位

```
侦察兵 (09:25)          狙击手 (09:30-15:00)        弹药库 (15:30)
─────────────────── → ────────────────────── → ──────────────────────
「哪些股值得看」         「现在该不该动手」           「收了盘算总账」
  资金流 + 竞价           持仓止损 + 入场信号           净值同步 + 风控体检
  → daily_pool.json      ← target_pool.json          → holdings.json
```

- **狙击手**：盘中实时。只做一件事——在当前时刻告诉你该不该买、该不该卖。秒级止损告警，不等人。
- **弹药库**：盘后风控。收盘后算总账——持仓市值同步、移动止盈上移、行业集中度检查、流动性风险评估、净值曲线更新。

## 二、狙击手 🎯 — 日中实时监控

### 2.1 双版本架构

| 维度 | v3.1 脚本版 | v4.0 守护进程版 |
|:--|:--|:--|
| 运行方式 | `python3 sniper.py` 手动触发 | systemd 服务，09:29 自动启动 |
| 轮询频率 | 单次扫描 | **每 3 秒** L1 持仓扫描 |
| 止损响应 | 取决于触发间隔 | **≤ 6 秒**（2 tick 确认） |
| 告警去重 | 无可复用状态 | 状态机跃迁 + 每只冷却 |
| 数据源 | Sina 批量 HTTP | Sina 批量 HTTP（同源） |
| 当前状态 | **保留备用** | **主力运行** (sniperd.service) |

> v4.0 守护进程详细设计 → `stock-research` → `references/sniper-v4-design.md`

### 2.2 四级信号体系

狙击手对**持仓中**的每只股票输出四级信号：

| 级别 | 触发条件 | 操作 | 响应速度 |
|:--|:--|:--|:--|
| **P0 🔴 止损触发** | `现价 ≤ max(固定止损, 移动止盈)` | **立即清仓** | 无冷却，2 tick 确认 |
| **P1 🟡 逼近止损** | `0 < 距止损 ≤ 3%` | **准备减仓** | 每只 120s 冷却 |
| **P2 🔵 显著异动** | `abs(涨跌) > 5%` 或 `量比 > 3x` | 大涨→移止盈 / 大跌→查利空 | 每只 300s 冷却 |
| **P3 ⚪ 常规** | 正常波动 | 持有/观察 | 不单独告警 |

**P2 详细判断逻辑**（`sniper.py:154-181`）：
- 大涨 +5% 且量比 > 2x → 建议**移动止盈上移**（锁定利润）
- 大跌 -5% → 建议**查基本面利空**（是否黑天鹅）
- 单纯放量 → 密切关注即可

**有效止损**取 `max(固定止损, 移动止盈价)`——哪个更高用哪个，确保利润不回吐。

### 2.3 入场信号 🆕 — 从目标池选买点

除了监控持仓，狙击手还从 **目标池**（`target_pool.json`，侦察兵开盘选出的 3 只优质标的）中判断是否该建仓。

入场信号三要素（`sniper.py:201-387`）：

```
量比分析               分时K线形态              议会信号
(_get_volume_ratio)  (_analyze_intraday_kline)  (target.parliament)
       ↓                     ↓                       ↓
  量比 > 1.5 放量      趋势: up/down/neutral     偏多/偏空/中立
  量比 > 2.0 强放量    形态: V反/连续上攻/回落
       ↓                     ↓
       └────── 综合判断 (_build_entry_signal) ──────┘
                         │
              分时ready + 量比>1.2 + 涨幅>-3%
                         │
                    🟢 入场信号
```

**分时K线技术分析**（拉取 48 根 5 分钟 K 线，覆盖约 4 小时）：

1. **趋势**：MA5 vs MA10 vs 现价 —— `up`（多头排列）/ `down`（空头排列）/ `neutral`
2. **形态识别**（近 3 根）：
   - `V型反转`：先跌后涨 → 资金抄底
   - `连续上攻`：3 根递涨 → 量价配合
   - `连续回落`：3 根递跌 → 回避
3. **量价关系**：近 3 根均量 > 整体均量 × 1.3 → 放量有效

入场就绪条件：`趋势=up` + 形态为 `V反/连续上攻` + `放量有效` + `量比>1.2` + `涨幅>-3%`

### 2.4 报告结构

```
🎯 狙击手 · 日内监控
时间: HH:MM | 净值: ¥xxx | P0:x P1:x | 入场:x只

### 🔴 P0 · 立即处理
  代码 名称 — 现价 ≤ 止损价，浮亏 x%
  操作: 立即清仓

### 🟡 P1 · 需要关注
  代码 名称 — 距止损 x%（现价 vs 止损价）
  操作: 准备减仓

### 持仓总览 (表格)
  代码 | 名称 | 现价 | 涨跌 | 信号 | 操作 | ⭐池标记

### 🎯 目标池入场
  🟢/⏳ 代码 名称  现价 涨跌%  量比:x  议会:bias  评分:xx
  > 入场理由：放量x倍 · 分时up/V反 · 议会偏多 → 🟢 放量突破·可建底仓

### 市场
  上证 xxxx +x% | 主力 +x亿
```

### 2.5 数据源

| 数据 | 来源 | 说明 |
|:--|:--|:--|
| 持仓实时价 | `get_stock_realtime()` Sina 批量 HTTP | 一次传全部 code |
| 目标池 | `target_pool.json` | 侦察兵写入 |
| 推荐池 | `daily_pool.json` | 推荐引擎写入，用于 ⭐ 标记 |
| 5日均量 | `quote.avg_volume_5` / `quote.volume_ratio` | 量比计算 |

| 大盘资金 | `get_market_money_flow()` | 主力净额 + 上证 |

## 三、弹药库 🛡️ — 盘后风控

### 3.1 核心职责

弹药库在**收盘后（15:30）**运行，确保持仓数据与实盘一致、风控参数更新到最新状态：

```
python3 ammo_risk.py --update    # 仅同步持仓（8项操作）
python3 ammo_risk.py             # 同步 + 生成风控报告
```

**Cron 自动调用**：`cron_ammo.sh → ammo_risk.py --update`（15:30）

### 3.2 八项同步操作 (`run_update()`)

```
run_update()
  │
  ├─ ① 市值同步 — 拉取最新价 → 更新每只持仓的 marketValue / unrealizedPnL / pnlPct
  │     数据来源：get_stock_realtime(codes) → Sina 批量 HTTP
  │
  ├─ ② 净值修正 — new_nv = 可用现金 + Σ市值 → 写入 accountInfo.currentNetValue
  │     同时同步到 rules.riskManagement.currentNetValue（兼容旧代码读取）
  │
  ├─ ③ 移动止盈 — 涨 20% 启动，每涨 10% 上移 10%
  │     动态缓冲：止损距现价至少保留 3%
  │
  ├─ ④ 回撤追踪 — 峰值净值 vs 当前净值 → currentDrawdown
  │
  ├─ ⑤ R值计算 — R值 = 净值 × 33.3% × 1/8 × 凯利(0.2)
  │     单笔最大可承受风险
  │
  ├─ ⑥ V8.0 池交叉标记 — 标记每只持仓是否在今日推荐池 (inPool)
  │
  ├─ ⑦ 净值历史追加 — netValueHistory 数组，保留最近 30 天
  │
  └─ ⑧ 保存 holdings.json
```

### 3.3 风控参数体系

| 参数 | 默认值 | 说明 | 位置 |
|:--|:--|:--|:--|
| 单股仓位上限 | 33.3% | maxPositionPerStock | rules |
| 总持仓上限 | 9 只 | maxHoldings | rules |
| R 值 | 动态 | 净值 × 33.3% × 0.125 × 0.2 | 每次 --update 重算 |
| 凯利系数 | 0.2 | kellyValue | rules.riskManagement |
| 移动止盈启动 | 涨 20% | initialTrigger | trailingStopLoss |
| 移动止盈步长 | 每涨 10% 上移 10% | triggerStep / moveStep | trailingStopLoss |
| 动态缓冲 | 止损距现价 ≥ 3% | 防过紧止损 | update_trailing_stops() |

### 3.4 行业集中度

```
分类方法: classify_industry(code, name)
  ├─ 主通道: tushare pro.stock_basic() → industry 字段（官方分类）
  └─ 回退:   名称关键词匹配 → 10 大行业组
             通信 / 新能源 / 半导体 / 电气设备 / IT服务
             食品饮料 / 医药生物 / 机械设备 / 基础化工 / 有色金属

告警规则:
  · 行业占比 > 30% → ⚡ 告警（建议 ≤ 30%）
  · 行业占比 > 15% → · 标记
```

### 3.5 流动性风险评估

检查持仓规模相对于市场流动性的冲击成本：

```
卖出 20% 持仓金额 ÷ 5日均成交额 = 冲击比例

冲击 > 5%  → 🟡 告警（可能推低价格）
冲击 > 10% → 🔴 严重（大额抛售困难）
```

5 日均成交额通过 `_get_avg_amount_5d()` 从 `data fetch CLI` 获取日线后计算。

### 3.6 组合层面风控 (v8.6+)

集成 `portfolio_risk.py`，弹药库报告末尾追加：

- **VaR (95%)**：在 95% 置信度下，组合一日最大亏损
- **CVaR (95%)**：超过 VaR 的尾部期望损失
- **历史最大回撤**：持仓组合历史最差表现
- **相关性告警**：持仓间相关性过高 → 降低分散化效果

### 3.7 报告结构

```
🛡️ 弹药库 · 收盘风控
日期: YYYY-MM-DD | 净值: ¥xxx | 总收益: ±x% | R值: ¥x | 回撤: x%

📊 大盘 — 主力 ±x亿 | 上证 xxxx (+x%)
📈 5日变化 — ±x% (¥prev → ¥now)

── 行业分布 ──
  行业 | 占比 | 标的
  ⚠️ xx  30%+ 告警

── 持仓明细 ──
  名称 代码 — 🟢/🔴 ¥现价 | 浮亏 ¥±x (±x%) ⭐池/⚡不在池
    股数 · 成本 · 仓位x%
    🎯 退出: 止损 ¥xx (距±x%)
    不在池告警 / 仓位超标告警 / 止损告警

── 流动性风险 ──
  卖出20%持仓约¥x万，占5日均量x万的x%

── 组合风控 ──
  VaR/CVaR/历史最大回撤

── 持仓汇总 ──
  可用资金 | 成本合计 | 市值合计 | 浮动盈亏 | 峰值净值
```

### 3.8 行情缓存机制

同一进程内 `run_update()` → `build_risk_report()` 复用行情数据：

```
get_cached_quotes(codes, force_refresh=True)  ← --update 时强制刷新
                     │
                     ├─ 缓存到全局 _quote_cache
                     │
get_cached_quotes(codes)                      ← --report 时复用
                     │
                     └─ 120s 内有缓存直接返回，不重复拉取
```
