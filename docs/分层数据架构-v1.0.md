## 二、历史数据仓库设计

### 时间维度索引

```
data/warehouse/
├── stock_kline/        ← 全A每日K线（按日期分区）
│   ├── 2026/06/03.parquet    (5500行 × 15列)
│   ├── 2026/06/02.parquet
│   └── ...
├── stock_fundamentals/  ← 全A每日基本面
│   ├── 2026/06/03.parquet    (5500行 × 12列)
│   └── ...
├── stock_factors/       ← 全A每日因子面板
│   ├── 2026/06/03_v3.parquet (5500行 × 35列)
│   └── ...
├── events/              ← 事件时间线
│   ├── announcements/2026/06/03.parquet
│   ├── limit_ups/2026/06/03.parquet
│   ├── block_trades/2026/06/03.parquet
│   └── ...
├── market_wide/         ← 市场级数据
│   ├── indices/2026/06/03.parquet
│   ├── fund_flows/2026/06/03.parquet
│   ├── northbound/2026/06/03.parquet
│   └── ...
└── _catalog/            ← 元数据
    ├── date_index.json       ← 日期→可用数据清单
    ├── stock_master.parquet  ← 股票主表(代码/名称/行业/上市日/退市日)
    └── schema_versions.json  ← 各层schema版本历史
```

### 文件格式选择

| 数据层级 | 格式 | 原因 |
|:--|:--|:--|
| Bronze | Parquet (snappy) | 列存+压缩，100万行K线 ~20MB |
| Silver | Parquet (snappy) | 同上，支持分区裁剪 |
| Gold | Parquet + JSON | 因子面板用Parquet，推荐结果用JSON |
| 元数据 | JSON | 人类可读，方便调试 |
| 模型 | .pkl / .npz | sklearn原生格式 |

### 数据血缘链

```

```

## 三、迁移路径

### 阶段 1: Bronze 层落地 (P0)

```
目标: 每个交易日收盘后，把所有原始数据冻结为不可变 Parquet

新建:
  data/bronze/                    ← 原始数据根目录
  data/bronze/_meta/              ← 采集清单+字段定义

改造:
  data_pipeline.py → 每次 get_* 调用时同步写 Bronze 层
  mega_collector.py → 每小时采集结果写 Bronze 层
  废弃 data/knowledge_base/      ← 合并到 Bronze events/

验收:
  python3 scripts/bronze_verifier.py --date 2026-06-03
  → "K线: 5280只, 基本面: 3913只, 事件: 156条, 完整性: 95.3%"
```

### 阶段 2: Silver 层建设 (P0)

```
目标: 从 Bronze 生成清洗后的统一数据

新建:
  data/silver/
  scripts/silver_pipeline.py     ← Bronze→Silver 的ETL

规则:
  停牌日: forward-fill close, volume=0
  复权: 统一前复权 (已有，BaoStock adjustflag='2')
  异常: close变化>20%且非涨跌停 → quality_flag='suspicious'
  去ST: stock_master 中标记 status=ST → 过滤

验收:
  从 Bronze 全量重建 Silver → 100% 一致
```

### 阶段 3: Gold 层 + 历史回填 (P1) ✅ 已落地

```
目标: Silver → factor_panel + ML datasets + daily_pool 归档

新建:
  data/gold/
  scripts/gold_pipeline.py       ← Silver→Gold 的因子计算 ✅
  scripts/gold_verifier.py       ← Gold 完整性验证 ✅
  scripts/cron_gold.sh           ← 每日 cron 15:50 ✅

改造:
  factor_evaluator.py → 从 Gold 层读因子，不再调 API (待迁移)
  stock_recommender.py → 推荐结果写 Gold daily_pool/ ✅
  ml_predictor.py → 训练数据从 Gold ml_datasets/ 读 (待迁移)

因子注册表 (v3, 26维):
  动量: mom_5d, mom_20d, mom_60d, alpha_idx
  波动: atr_14, vol_20d, downside_vol
  资金: net_flow, turnover, volume_ratio
  筹码: turnover_zscore, vol_ratio_trend, amplitude_5d
  估值: pe_ttm, pb, total_mv_log
  质量: is_suspended, is_st, n_quality_flags
  滚动: close_zscore, volume_zscore, pe_percentile, pb_percentile, ma5_deviation, ma20_deviation

启动期说明:
  - 因子覆盖率从 ~24% 起步 (仅有资金+质量类可直接从 Silver 读取)
  - 随 Silver 历史积累 (60日+)，动量/波动/筹码/滚动因子自动补全至 100%
  - Gold 主读 Silver，启动期补充读 Bronze 历史日线
```

## 四、可复现性保证

### 每日快照命令

```bash
# 完整重放: 从 Bronze 重建当天的 Silver + Gold
python3 scripts/replay_day.py --date 2026-06-03

# 输出:
#   [Bronze] 读取 5280 K线 + 3913 基本面 + 156 事件
#   [Silver] 清洗完成: 5280行, 质量标记 12条异常
#   [Gold]   因子面板 v3: 35因子 × 5280行
#   [Gold]   ML训练集: 85000 样本
#   [Verify] Silver hash = a1b2... (与当日一致 ✅)
#   [Verify] Gold hash   = c3d4... (与当日一致 ✅)
```
