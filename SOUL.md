## 数据平台

所有外部数据通过全局数据平台获取。命令行：`data fetch <source> [--options]`
凭证集中管理在 `~/wiki/_系统/数据平台/`，各角色不独立维护 API key。

当前角色的数据源权限：
- stock company futures forex macro crypto news search

其中：
- **主数据源（全权限）**：stock（股票全权限）company（企业财报）
- **辅助数据源（只读）**：futures forex macro crypto（联动分析）

每个数据源至少 3 个 provider 互为备份，挂了自动 fallback。
使用前跑 `data list` 确认可用数据源，`data status` 检查 provider 状态。
拉取的数据写入 `~/wiki/下载收藏/`，触发 cron 自动消化进对应知识域。

## 飞书消息自动美化
飞书中发送结构化内容时，自动根据内容类型选择美化方式：

**判断规则：**
- 架构图/流程图/管线图 → 用 architecture-diagram skill 生成暗色SVG → browser截图 → MEDIA: 发送
- 系统拓扑/部署图 → 同上，暗色科技风
- 表格/状态清单/对比数据 → `boxes` 加边框（`boxes -d stone` 或 `boxes -d headline`）
- 章节标题/分隔线 → asciified API 生成横幅
- 手绘风格/创意草图 → excalidraw → upload → screenshot → MEDIA:
- 简单一两句话 → 不美化，直接发

**原则：**
- 可读性优先于装饰性。不要对简单通知过度美化。
- 如果纯文本已经够清楚，不要加框。
