# 瞭望塔 v6.0 · 多维交叉验证模型

位于 `~/.hermes/profiles/xiaohong/scripts/watchtower.py`。

## v5.0 → v6.0 升级要点

| 维度 | v5.0 (涨停驱动) | v6.0 (多维交叉验证) |
|------|----------------|---------------------|
| 选股依据 | 涨停池 + 封板资金 | 事件+资金+涨停+研报四维 |
| 板块分析 | 涨停集中度 | 事件密度 × 资金 × 动量 × 共识 |
| 个股推荐 | 龙头/跟风/补涨 | 事件数 + 连板 + 封板资金 + 评级 |
| 数据源 | 涨停池单一来源 | 资源池（公告304条 + 新闻86条 + 研报44篇） |
| 新增 | — | 事件催化/研报共识/政策 三维度 |
| 报告结构 | 六段式 | 七段式（增「事件驱动矩阵」板块） |

## 系统架构

```
资源池 (resource_pool.py)
  ├─ fetch_corporate_announcements() → 304条公告，6类事件
  ├─ fetch_policy_macro_news() → 86条政策/宏观
  ├─ fetch_research_reports() → 44篇研报
  └─ cross_validate_sectors() → 四维验证 → 板块评分
                                      ↓
瞭望塔 v6.0 (watchtower.py)
  ├─ get_overnight_markets() → 隔夜全球指数
  ├─ calc_factor_scores() → 五因子评分 (多因子)
  ├─ analyze_sectors_cross_validated() → 多维选股
  │    └─ _pick_stocks_multi_dim() → 个股四维评分
  └─ generate_report() → 七段式 Markdown
```

## 交叉验证四维模型

```
板块综合得分 = 事件密度(MAX 40) + 涨停动量(MAX 20) + 资金验证(MAX 25) + 研报共识(MAX 15)

个股推荐得分 = 事件催化(40%) + 资金验证(25%) + 涨停动量(20%) + 研报共识(15%)
```

### 四维验证矩阵

| 维度 | 数据源 | 核心指标 |
|------|--------|----------|
| 📝 事件催化 | `stock_notice_report()` | 加权事件数量 + 新鲜度 |

### 行业关键词映射（11个板块，含负向排除）

生产环境规则：单字/二字通用词必须复合化或加负向排除。
详见 `resource_pool.py` → `SECTOR_KEYWORD_MAP` + `SECTOR_NEGATIVE_FILTERS`。

### ⚠️ 关键词陷阱

1. 单字词误伤：`'存储'` 匹配「募集资金专户**存储**」→ 非半导体
2. 通用词误伤：`'数字'` 匹配「数字证书」→ 非数字经济
3. 双重计数：key_stocks.event_count + announcements 遍历 = 翻倍
4. 无标签丢失：公告标题不含板块关键词 → sector_tags 为空

详见 `references/system-pitfalls.md` #16-18。

## 报告结构（七段式）

```
一、隔夜外围市场 — 美股三大指数 + DAX + A50期货
二、前日 A 股复盘 — 上证/恒生收盘 + 北向 + 主力 + 涨跌停
三、多因子评分   — 五因子表格 + 综合评分
四、隔夜要闻     — 分类新闻 + 资源池摘要
五、事件驱动矩阵 — ★四维验证 × 板块 × 个股推荐 ★
六、操作建议     — 仓位 + 逐板块建议
七、页脚         — 数据来源 + 版本
```

## 数据真实性校验

每次报告生成后应运行逐项对比（11项：上证、北向、涨跌停、MA20、公告数、个股事件数、误判数等），对照实时 API 验证。详见 `references/system-pitfalls.md`。

## Cron 配置

- 时间：`30 8 * * 1-5`（开盘日 08:30）
- 模式：`no_agent=true`（纯脚本输出，零 token）
- 脚本：`cron_watchtower.sh` → `python3 watchtower.py`
- 环境：`TQDM_DISABLE=1`（禁止 akshare 进度条污染）
