## 1. 产品概述

### 1.1 产品定位

小红量化交易系统是一款 **AI 驱动的全自动量化交易 SaaS 平台**，面向个人投资者和中小型投资工作室，覆盖 A 股市场，提供从「数据采集 → 因子挖掘 → 选股推荐 → 盘中监控 → 自动下单 → 复盘优化」的**全闭环量化交易能力**。

与传统量化平台（聚宽、米筐）不同，小红不是回测工具，不是手动信号工具——而是一个**装好就能跑的 AI 交易员**：用户完成注册和券商绑定后，系统自动运转，每日推送选股建议、盘中实时监控、自动执行交易、收盘后自动复盘。

### 1.2 核心价值主张

| 维度 | 用户痛点 | 小红方案 |
|------|----------|----------|
| **学习门槛** | 量化交易需要编程 + 金融 + 数学三重技能 | 零代码配置，策略模板一键启用 |
| **时间投入** | 手动盯盘占用大量时间 | AI 全天候自动监控，事件驱动告警 |
| **情绪干扰** | 贪婪/恐惧导致非理性决策 | 规则引擎严格执行，不情绪化 |
| **策略迭代** | 手动回测效率低、参数调优靠猜 | 自动化因子评估 + ML 模型 + 进化引擎 |
| **券商对接** | 多券商 API 互不兼容 | 统一网关，一套接口对接全部券商 |
| **成本控制** | 自建系统需要服务器 + 数据 + AI 成本 | SaaS 模式，按月付费，零运维 |

### 1.3 产品边界

- **支持市场**：A 股（沪深北），后续扩展港股、美股
- **交易品种**：股票（含 ETF），后续扩展可转债
- **策略类型**：中低频（日频 + 盘中小时级），不涉及高频交易
- **部署方式**：SaaS 云平台 + 桌面客户端（Pro 版可本地运行）
- **用户规模**：设计目标支持 10,000 并发用户

## 2. 市场分析

### 2.1 市场规模

中国量化交易市场处于快速增长期：

| 指标 | 数据 |
|------|:--:|
| A 股个人投资者数量 | 2.2 亿 |
| 量化私募管理规模 | 1.5 万亿元 |
| 量化交易占比（A股成交额） | ~25% |
| 个人量化工具市场规模（预估） | 50-100 亿元/年 |

### 2.2 竞品分析

| 产品 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| 聚宽 (JoinQuant) | 在线回测 | 社区活跃、数据丰富 | 无实盘交易、无 AI 认知层 |
| 米筐 (RiceQuant) | 在线回测 | 数据质量高、机构客户 | 门槛高、需编程 |
| 掘金量化 | 终端 | 低延迟、券商合作 | 需下载客户端、无 AI |
| TradingSkill | 开源工具 | 免费、可定制 | 无 UI、需自行部署 |
| Vibetrading | AI 策略 | LLM 驱动探索 | 无实盘执行 |
| QMT（券商自带） | 券商工具 | 直接交易 | 策略需自建、无 AI |
| **小红（本产品）** | **AI 量化 SaaS** | **零代码 + AI 认知 + 券商对接 + 全自动** | 品牌新、需建立信任 |

### 2.3 竞争优势

1. **AI 认知层差异化**：LLM 驱动的市场研判、选股逻辑解读、自动复盘，不是冷冰冰的信号列表
2. **全闭环**：从选股到下单到复盘，一个平台完成全流程
3. **券商统一网关**：适配多券商 API（QMT / Easytrader / CTP），用户无需关心底层差异
4. **零代码策略配置**：可视化因子权重调节 + 策略模板市场，降低使用门槛
5. **可进化**：系统根据实盘表现自动优化参数，策略越用越准

## 4. 产品定义

### 4.1 产品架构

小红采用**三层架构**：

```
┌─────────────────────────────────────────────────┐
│              用户界面层 (Presentation)             │
│  Web SPA  │  微信小程序  │  消息 Bot  │  REST API  │
├─────────────────────────────────────────────────┤
│              平台服务层 (Platform)                 │
│  用户中心  │  策略引擎  │  通知中心  │  订阅管理    │
├─────────────────────────────────────────────────┤
│              核心引擎层 (Core Engine)              │
│  数据管线  │  选股引擎  │  盘中监控  │  执行引擎    │
│  AI 认知  │  风控引擎  │  回测系统  │  复盘引擎    │
└─────────────────────────────────────────────────┘
```

### 4.2 核心功能全景

```
小红量化交易系统
├── 1. 数据管线
│   ├── 多源实时行情采集
│   ├── 历史数据 ETL（日K / 分钟K / 分时）
│   ├── 基本面数据（财报 / 估值 / 研报）
│   ├── 另类数据（资金流向 / 龙虎榜 / 公告）
│   └── 分层存储（Bronze → Silver → Gold）
│
├── 2. 选股引擎
│   ├── 多因子打分模型（事件/资金/情绪/技术/基本面/另类）
│   ├── AI 研究员深度分析
│   ├── 策略模板市场（均衡/进取/防御/行业专项）
│   ├── ML 涨跌预测
│   └── 每日推荐池生成
│
├── 3. 盘中监控
│   ├── 侦察兵：资金异动扫描 + 技术面确认
│   ├── 狙击手：实时价格 + 止损秒级响应
│   ├── 竞价分析：集合竞价信号采集与学习
│   └── 盘中推荐池动态更新
│
├── 4. 交易执行
│   ├── 券商统一网关（Paper / QMT / Easytrader）
│   ├── 订单管理（TWAP / VWAP 算法拆单）
│   ├── 风控前置检查（仓位/资金/频率/涨跌停）
│   └── 交易审计日志
│
├── 5. 风控管理
│   ├── 多层级风控（单票/行业/组合/回撤）
│   ├── VaR / CVaR 风险度量
│   ├── 压力测试（5 场景）
│   ├── 相关性矩阵监控
│   └── 移动止盈 + R 值约束
│
├── 6. 复盘优化
│   ├── 日度复盘报告（收益归因 + 选股复盘）
│   ├── 周度深度复盘（行为偏差 + 叙事性分析）
│   ├── 因子有效性评估（IC / ICIR）
│   ├── 进化引擎（LLM 驱动参数自动优化）
│   └── 权益曲线 + 回撤曲线可视化
│
├── 7. AI 认知层
│   ├── 瞭望塔晨报：每日市场研判 + 操作策略
│   ├── 决策官：盘中综合决策
│   ├── 文工团：每日复盘诊断
│   └── LLM 解耦：支持 Claude / GPT / DeepSeek / 本地模型
│
└── 8. 用户界面
    ├── Web 管理后台（策略配置/持仓管理/报表）
    ├── 微信小程序（查看持仓/接收信号/读日报）
    ├── 消息推送（企微/钉钉/飞书/邮件/短信）
    └── REST API（第三方集成）
```

### 6.2 数据流架构

市场的原始数据经过 **三层分层处理**，逐层增值：

```
外部数据源                        Bronze 层               Silver 层              Gold 层
═══════════                      ═══════════              ═══════════            ══════════
                                 (不可变原始)              (清洗统一)              (特征就绪)
CHOICE / Wind ─────┐
tushare ───────────┤
AKShare ───────────┼──→  raw JSON.gz  ──→  标准化表  ──→  因子面板
东方财富 API ──────┤     按日分区          全A统一格式       26维因子
交易所行情 ────────┘     不可篡改          质量标记          ML数据集
                                                        
                        数据血缘可追溯      可复现           下游消费就绪
```

### 6.3 分层数据架构详解

#### Bronze 层（不可变原始数据）

- **定位**：数据湖第一站，原始格式冻结，永不修改
- **存储**：gzip 压缩 JSON，按日期 + 数据类型分区
- **数据类**：行情、资金流、基本面、事件、指数
- **保留策略**：热数据 1 年，冷数据归档到对象存储

#### Silver 层（清洗统一数据）

- **定位**：Bronze → 清洗 → 统一 Schema → 质量标记
- **存储**：Parquet 列存，按日期分区
- **覆盖**：全市场 5,500+ 只 A 股
- **质量标记**：每行数据附 `_quality` 字段（数据源/时效/完整性）

#### Gold 层（特征因子数据）

- **定位**：Silver → 因子计算 → 建模就绪
- **产出**：因子面板、ML 数据集、推荐池快照
- **因子**：动量/波动/质量/估值/资金/情绪/筹码/另类 八大类
- **消费方**：选股引擎、回测系统、报告系统

## 8. 券商对接方案

### 8.1 券商驱动标准接口

所有券商通过统一的 `BrokerDriver` 抽象接口接入：

```python
class BrokerDriver(ABC):
    """券商驱动标准接口"""

    # 连接管理
    def connect(self, config: BrokerConfig) -> bool

    # 账户与持仓
    def get_account(self) -> Account
    def get_positions(self) -> List[Position]

    # 订单管理
    def submit_order(self, order: OrderRequest) -> OrderResponse
    def cancel_order(self, order_id: str) -> bool
    def get_order(self, order_id: str) -> Order
    def query_orders(self, date: date) -> List[Order]

    # 行情
    def subscribe_quote(self, symbols: List[str]) -> None
    def get_quote(self, symbol: str) -> Quote
```

| 夏普比率 | 风险调整后收益 = (收益 - 无风险利率) / 波动率 |
| QMT | 券商提供的量化交易终端和 API |
| Easytrader | 开源券商对接库，支持华泰/银河等 |
| CTP | 期货/股票综合交易平台，低延迟 |
| TWAP | 时间加权平均价格算法下单 |
| VWAP | 成交量加权平均价格算法下单 |
| VaR | 在险价值，衡量最大可能亏损 |
| CVaR | 条件在险价值，超过 VaR 的平均亏损 |
| Bronze/Silver/Gold | 数据分层架构的三层命名 |
| RLS | PostgreSQL 行级安全 |

### B. 关键决策记录

| 决策 | 结论 | 理由 |
|------|------|------|
| 券商优先方案 | QMT > Easytrader > CTP | QMT 券商支持最广，SDK 最完善 |

| 开发语言 | Python | 量化生态最丰富，团队技能匹配 |
